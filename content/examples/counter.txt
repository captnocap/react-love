=== OVERVIEW ===
This example builds an interactive counter with two buttons. It demonstrates React state
management (useState), user interaction (Pressable with render props for press feedback),
row layout (flexDirection: 'row' with explicit width), and the template literal pattern
for dynamic text content.

=== CODE ===
---
import { useState } from 'react';
import { Box, Text, Pressable } from '@ilovereact/core';

function Button({ label, onPress }: { label: string; onPress: () => void }) {
  return (
    <Pressable onPress={onPress}>
      {({ pressed }) => (
        <Box style={{
          width: 64,
          height: 64,
          backgroundColor: pressed ? '#1d4ed8' : '#3b82f6',
          borderRadius: 12,
          justifyContent: 'center',
          alignItems: 'center',
        }}>
          <Text style={{ fontSize: 28, color: '#ffffff', fontWeight: 'bold' }}>
            {label}
          </Text>
        </Box>
      )}
    </Pressable>
  );
}

export default function App() {
  const [count, setCount] = useState(0);

  return (
    <Box style={{
      width: '100%',
      height: '100%',
      backgroundColor: '#0f172a',
      justifyContent: 'center',
      alignItems: 'center',
      gap: 32,
    }}>
      <Text style={{ fontSize: 64, color: '#f8fafc' }}>
        {`${count}`}
      </Text>
      <Box style={{
        flexDirection: 'row',
        width: '100%',
        justifyContent: 'center',
        gap: 20,
      }}>
        <Button label="-" onPress={() => setCount(c => c - 1)} />
        <Button label="+" onPress={() => setCount(c => c + 1)} />
      </Box>
    </Box>
  );
}
---

=== EXPLANATION ===
1. The Button component uses Pressable with a render prop: ({ pressed }) => ...
   This gives access to the pressed state for visual feedback (darker blue when
   pressed).

2. Each button has fixed width: 64, height: 64 for a square shape. The buttons
   use borderRadius: 12 for rounded corners.

3. The counter display uses a template literal: {`${count}`}. This is important.
   Do NOT write {count} as it creates a number child node. Always wrap dynamic
   content in template literals for Text nodes.

4. The button row has flexDirection: 'row' and width: '100%'. The width is critical:
   without it, justifyContent: 'center' has no effect because the row Box has no
   known width to distribute space within.

5. gap: 20 adds 20 pixels of space between the two buttons.

6. gap: 32 on the root container adds space between the counter text and the
   button row.

7. fontWeight: 'bold' on the button text triggers faux-bold rendering (the text
   is drawn twice with a slight offset on Love2D).
