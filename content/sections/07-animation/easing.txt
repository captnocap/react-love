=== OVERVIEW ===
iLoveReact provides a set of easing functions that control the rate of change during
timing-based animations. Easing functions take a progress value t (0 to 1) and return a
transformed value that determines the animation's velocity curve. They are used with
AnimatedValue.timing() and useTransition().

The library includes standard easing curves (linear, easeIn, easeOut, easeInOut), a cubic
bezier function for custom curves, a bounce effect, and an elastic overshoot effect. All
easing functions conform to the EasingFunction type: (t: number) => number.

=== API / SYNTAX ===
Type:
  type EasingFunction = (t: number) => number

  Input: t is the linear progress from 0 (start) to 1 (end).
  Output: the eased progress value (may overshoot 0-1 range for elastic/bounce effects).

Static easing functions:

  Easing.linear
    (t) => t
    Constant velocity. No acceleration or deceleration.

  Easing.easeIn
    (t) => t * t
    Quadratic ease-in. Starts slow, accelerates.

  Easing.easeOut
    (t) => t * (2 - t)
    Quadratic ease-out. Starts fast, decelerates.

  Easing.easeInOut
    (t) => t < 0.5 ? 2 * t * t : -1 + (4 - 2 * t) * t
    Quadratic ease-in-out. Starts slow, accelerates, then decelerates.
    This is the default easing for AnimatedValue.timing().

  Easing.bounce
    Standard bounce curve. Simulates a ball bouncing and settling.
    Uses four quadratic segments with decreasing amplitude.

Factory easing functions (return an EasingFunction):

  Easing.bezier(x1: number, y1: number, x2: number, y2: number): EasingFunction
    Creates a cubic bezier easing function. The four parameters define two control points
    (P1 and P2) of a cubic bezier curve from (0,0) to (1,1). Uses Newton-Raphson iteration
    (8 iterations) to solve the cubic for the given t value.

    Common presets:
      Easing.bezier(0.25, 0.1, 0.25, 1)     // CSS ease
      Easing.bezier(0.42, 0, 1, 1)           // CSS ease-in
      Easing.bezier(0, 0, 0.58, 1)           // CSS ease-out
      Easing.bezier(0.42, 0, 0.58, 1)        // CSS ease-in-out
      Easing.bezier(0.16, 1, 0.3, 1)         // Expo ease-out
      Easing.bezier(0.34, 1.56, 0.64, 1)     // Back ease-out (overshoot)

  Easing.elastic(bounciness?: number): EasingFunction
    Creates an elastic easing that overshoots the target and oscillates before settling.
    Default bounciness: 1. Higher values = more oscillation. Lower values = less overshoot.
    Uses exponential decay with sinusoidal oscillation.

=== EXAMPLES ===
Example 1: Different easing curves
---
import { useEffect } from 'react';
import { useAnimation, Easing, Box } from '@ilovereact/core';

function EasingDemo() {
  const [linearAnim, linearX] = useAnimation(0);
  const [easeInAnim, easeInX] = useAnimation(0);
  const [easeOutAnim, easeOutX] = useAnimation(0);
  const [bounceAnim, bounceX] = useAnimation(0);

  useEffect(() => {
    linearAnim.timing({ toValue: 300, duration: 1000, easing: Easing.linear }).start();
    easeInAnim.timing({ toValue: 300, duration: 1000, easing: Easing.easeIn }).start();
    easeOutAnim.timing({ toValue: 300, duration: 1000, easing: Easing.easeOut }).start();
    bounceAnim.timing({ toValue: 300, duration: 1000, easing: Easing.bounce }).start();
  }, [linearAnim, easeInAnim, easeOutAnim, bounceAnim]);

  return (
    <Box style={{ width: '100%', height: '100%', padding: 20, gap: 16 }}>
      <Box style={{ width: 40, height: 40, backgroundColor: '#89b4fa', transform: { translateX: linearX } }} />
      <Box style={{ width: 40, height: 40, backgroundColor: '#a6e3a1', transform: { translateX: easeInX } }} />
      <Box style={{ width: 40, height: 40, backgroundColor: '#f38ba8', transform: { translateX: easeOutX } }} />
      <Box style={{ width: 40, height: 40, backgroundColor: '#f9e2af', transform: { translateX: bounceX } }} />
    </Box>
  );
}

Example 2: Custom bezier curve
---
import { useEffect } from 'react';
import { useAnimation, Easing, Box } from '@ilovereact/core';

function CustomEase() {
  const [anim, value] = useAnimation(0);

  useEffect(() => {
    anim.timing({
      toValue: 1,
      duration: 800,
      easing: Easing.bezier(0.34, 1.56, 0.64, 1), // overshoot then settle
    }).start();
  }, [anim]);

  return (
    <Box style={{
      width: 100,
      height: 100,
      backgroundColor: '#cba6f7',
      transform: { scaleX: value, scaleY: value },
    }} />
  );
}

Example 3: Elastic animation
---
import { useEffect } from 'react';
import { useAnimation, Easing, Box } from '@ilovereact/core';

function ElasticBox() {
  const [anim, value] = useAnimation(0);

  useEffect(() => {
    anim.timing({
      toValue: 200,
      duration: 1200,
      easing: Easing.elastic(1.5),
    }).start();
  }, [anim]);

  return (
    <Box style={{
      width: 60,
      height: 60,
      backgroundColor: '#f38ba8',
      borderRadius: 30,
      transform: { translateX: value },
    }} />
  );
}

=== PLATFORM NOTES ===
Easing functions are pure math -- they work identically on all platforms. The functions
execute in JavaScript (QuickJS on Love2D, V8/Node on other targets) and are evaluated
every animation frame.

=== CRITICAL RULES ===
- Easing functions must accept 0 and return 0, and accept 1 and return 1
- Easing.easeInOut is the default when no easing is specified in timing()
- Easing.bezier uses Newton-Raphson iteration (8 iterations) -- extremely fast but not exact for extreme curves
- Easing.elastic and Easing.bounce may produce values outside the 0-1 range (overshoot)
- Easing.elastic(0) will produce extreme oscillation -- use values >= 0.001
- All easing functions are stateless and can be shared across animations
