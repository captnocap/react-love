=== OVERVIEW ===
The web target (@ilovereact/web) renders iLoveReact components as real DOM elements with
CSS flexbox. The same Box, Text, and Image components that render as draw calls on
Love2D render as div, span, and img elements in the browser. The RendererMode context
switches the primitives between 'web' and 'native' behavior.

The web target can run standalone (a regular React web app) or alongside a Love2D canvas
(as an overlay for web builds of Love2D games using Emscripten).

=== API / SYNTAX ===
esbuild Configuration:
  --format=esm

Element Mapping:
  <Box>       -> <div> with CSS flexbox
  <Text>      -> <span> with inline display
  <Image>     -> <img> with block display
  <Pressable> -> <div> with pointer events, tabIndex, role="button"

Style Conversion (styleToCSS function):
  Style.justifyContent 'start'/'end'  -> CSS 'flex-start'/'flex-end'
  Style.color [r,g,b,a] (0-1 range)  -> CSS rgba(R,G,B,A) (0-255 range)
  Style.backgroundGradient            -> CSS linear-gradient()
  Style.transform                     -> CSS transform string
  Style.transition                    -> CSS transition property
  Style.shadowColor/Offset/Blur      -> CSS box-shadow
  Style.textShadowColor/Offset       -> CSS text-shadow
  Style.outlineWidth/Color/Offset    -> CSS outline

=== EXAMPLES ===
Example 1: Web-Only App
---
import React from 'react';
import { createRoot } from 'react-dom/client';
import { RendererProvider } from '@ilovereact/core';
import App from './App';

const root = createRoot(document.getElementById('root')!);
root.render(
  <RendererProvider mode="web">
    <App />
  </RendererProvider>
);
---
Platforms: web

Example 2: Web Overlay with Love2D Canvas
---
import { useLoveOverlays } from '@ilovereact/core';

function GameOverlay({ canvasRef }) {
  const overlays = useLoveOverlays(canvasRef);

  return (
    <div style={{ position: 'absolute', top: 0, left: 0 }}>
      {overlays.map(o => (
        <div key={o.id} style={{
          position: 'absolute',
          left: o.domX,
          top: o.domY,
        }}>
          {/* DOM overlay positioned relative to Love2D coordinates */}
        </div>
      ))}
    </div>
  );
}
---
Platforms: web

=== PLATFORM NOTES ===
Features:
  - Full CSS flexbox (browser-native layout engine)
  - CSS transitions and animations (mapped from Style.transition)
  - Hover/active styles via React state (onPointerEnter/Leave, onPointerDown/Up)
  - Drag support via pointer capture
  - Keyboard accessibility (tabIndex on interactive elements)
  - All DOM event types supported

Differences from Love2D:
  - Layout is computed by the browser, not layout.lua
  - Text measurement uses browser font metrics, not Love2D font:getWrap()
  - The browser applies its own defaults (margins on body, etc.)
  - Some visual properties may render slightly differently
