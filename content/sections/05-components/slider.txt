=== OVERVIEW ===
Slider provides a draggable input for selecting a numeric value within a range. It renders a track with an active fill and a draggable thumb. Slider supports both controlled (value prop) and uncontrolled (defaultValue prop) modes, step snapping, vertical orientation, and full visual customization of track and thumb colors and sizes.

In web mode, Slider handles drag via mouse events with global listeners for reliable tracking outside the element bounds. In native mode, it uses the Box drag event system (onDragStart, onDrag, onDragEnd) for pointer tracking. The native renderer uses a three-segment flex layout (active track, thumb, inactive track) rather than absolute positioning for compatibility with the Lua layout engine.

=== API / SYNTAX ===
```tsx
import { Slider } from '@ilovereact/core';
```

```tsx
const [volume, setVolume] = useState(0.5);

<Slider
  value={volume}
  onValueChange={setVolume}
  minimumValue={0}
  maximumValue={1}
  step={0.1}
  style={{ width: 200 }}
/>
```

Props:
| Prop             | Type                          | Default     | Description                                |
|------------------|-------------------------------|-------------|--------------------------------------------|
| value            | number                        | undefined   | Controlled value (overrides internal state)|
| defaultValue     | number                        | 0           | Initial value for uncontrolled mode        |
| onValueChange    | (value: number) => void       | undefined   | Called on every value change during drag    |
| onSlidingStart   | () => void                    | undefined   | Called when drag begins                    |
| onSlidingEnd     | (value: number) => void       | undefined   | Called when drag ends with final value     |
| minimumValue     | number                        | 0           | Minimum allowed value                      |
| maximumValue     | number                        | 1           | Maximum allowed value                      |
| step             | number                        | undefined   | Snap value to step increments              |
| disabled         | boolean                       | false       | Disable interaction                        |
| style            | Style                         | undefined   | Container style (width sets track length)  |
| trackColor       | Color                         | '#333333'   | Inactive track color                       |
| activeTrackColor | Color                         | '#4A90D9'   | Active (filled) track color                |
| thumbColor       | Color                         | '#ffffff'   | Thumb color                                |
| thumbSize        | number                        | 20          | Thumb diameter in pixels                   |
| trackHeight      | number                        | 4           | Track height in pixels                     |
| vertical         | boolean                       | false       | Vertical orientation                       |

---
```tsx
<Slider
  defaultValue={0.5}
  minimumValue={0}
  maximumValue={1}

```

=== PLATFORM NOTES ===
Love2D:
  Native mode uses a three-segment flex row layout: active track segment, thumb circle, inactive track segment. This avoids position: absolute which has limitations in the Lua layout engine for this use case. Drag tracking uses the Box drag event system (onDragStart, onDrag, onDragEnd) with totalDeltaX/totalDeltaY from the LoveEvent. Track width defaults to 200px if not specified via style.width. The value-at-drag-start is captured and deltas are applied relative to it for smooth, predictable dragging.

Web:
  Uses absolute positioning for the active track fill and thumb within the track container. Mouse events use global document listeners (mousemove, mouseup) during drag for reliable tracking even when the cursor leaves the slider bounds. Disabled state shows cursor: not-allowed on the thumb.

Terminal:
  Not available on grid targets. For value selection on character-grid renderers, build a custom component using keyboard input.

=== CRITICAL RULES ===
- In controlled mode (value prop provided), you MUST update the value in onValueChange. If you do not, the thumb will not move because the component re-renders with the stale controlled value.
- The style.width prop (or style.height for vertical) determines the track length. Without it, the slider defaults to 200px. Always set an explicit size for predictable layout.
- step snapping rounds to the nearest step increment. A step of 0.1 with range 0-1 produces values: 0, 0.1, 0.2, ..., 1.0. The snapped value is clamped to [minimumValue, maximumValue].
- onValueChange fires on every drag movement (potentially many times per second). Do not perform expensive operations in this callback. Use onSlidingEnd for final-value operations like network requests or heavy state updates.
- onSlidingEnd fires once when the drag gesture completes. Its argument is the final value after step snapping and clamping.
- Disabled sliders render at 50% opacity and ignore all interaction.
