=== OVERVIEW ===
Text is the primitive for rendering text content. It is the only element that carries intrinsic width -- the layout engine measures the text string against the specified font and fontSize to determine how much horizontal space it occupies. This makes Text unique among primitives: it is the only element that can size itself without an explicit width prop.

Text supports font styling (fontSize, fontFamily, fontWeight, color, textAlign, letterSpacing, lineHeight), text truncation via numberOfLines, and keyboard event handlers. In web mode it renders as a `<span>`; in native mode it outputs a `'Text'` host element for the Lua painter.

=== API / SYNTAX ===
```tsx
import { Text } from '@ilovereact/core';
```

```tsx
<Text style={{ fontSize: 16, color: '#cdd6f4' }}>
  Hello, world
</Text>
```

Props:
| Prop          | Type                         | Default   | Description                              |
|---------------|------------------------------|-----------|------------------------------------------|
| style         | Style                        | undefined | Text styling (fontSize, color, etc.)     |
| numberOfLines | number                       | undefined | Truncate text to N lines with ellipsis   |
| onKeyDown     | (event: LoveEvent) => void   | undefined | Key pressed while focused                |
| onKeyUp       | (event: LoveEvent) => void   | undefined | Key released while focused               |
| onTextInput   | (event: LoveEvent) => void   | undefined | Text input while focused                 |
| children      | ReactNode                    | undefined | Text content (strings, numbers, etc.)    |

Relevant Style properties for Text:
| Property           | Type                              | Description                              |
|--------------------|-----------------------------------|------------------------------------------|
| fontSize           | number                            | Font size in pixels (REQUIRED)           |
| color              | Color                             | Text color                               |
| fontFamily         | string                            | Font family name                         |
| fontWeight         | 'normal' | 'bold' | number        | Font weight                              |
| textAlign          | 'left' | 'center' | 'right'       | Horizontal text alignment                |
| textOverflow       | 'clip' | 'ellipsis'               | How overflowing text is handled          |
| textDecorationLine | 'none' | 'underline' | 'line-through' | Text decoration                    |
| lineHeight         | number                            | Line height in pixels                    |
| letterSpacing      | number                            | Extra space between characters            |
| textShadowColor    | Color                             | Text shadow color                        |
| textShadowOffsetX  | number                            | Text shadow horizontal offset            |
| textShadowOffsetY  | number                            | Text shadow vertical offset              |

Example 3: Truncated text with numberOfLines
---
```tsx
<Box style={{ width: 200 }}>
  <Text style={{ fontSize: 14, color: '#a6adc8' }} numberOfLines={2}>
    This is a long paragraph of text that will be truncated to two lines
    with an ellipsis at the end if it overflows the available space.
  </Text>
</Box>
```
---
Platforms: All

---

```tsx
</Text>
```

Platforms: All

=== PLATFORM NOTES ===
Love2D:
  Renders as a 'Text' host element. Font measurement happens in Lua via love.graphics.getFont():getWidth(). The painter draws text with love.graphics.print(). Custom fonts are loaded by fontFamily name from the project's font directory. Text wrapping is computed by the layout engine when the parent constrains width.

Web:
  Renders as a <span> with display: inline. The styleToCSS() converter maps fontSize, color, fontFamily, and other text properties to their CSS equivalents. numberOfLines uses -webkit-line-clamp for multi-line truncation with ellipsis.

Terminal:
  Grid targets use <text> lowercase intrinsic. Character-grid rendering means fontSize has no effect; text occupies cells based on character count. Color maps to ANSI truecolor or palette indices depending on the target.

=== CRITICAL RULES ===
- EVERY Text element MUST have an explicit fontSize in its style prop. The static linter (`ilovereact lint`) enforces this as a build-blocking error. Text without fontSize will produce unpredictable rendering because the layout engine cannot measure the text.
- Do not mix bare text and JSX expressions: `"count: {value}"` renders literally as that string. Use template literals instead: `{`count: ${value}`}`.
- Text is the only primitive with intrinsic width. Box and Image have zero width unless you set one explicitly. This means Text can size itself and its parent, but Box cannot.
- Do not nest Box inside Text. Text children should be strings, numbers, or other Text elements only.
- Do not use the block character (U+2588) inside Text. It renders as a font glyph, not a filled pixel. Use colored Box elements with backgroundColor instead. The linter enforces this via `no-block-char-in-text`.
