=== OVERVIEW ===
This guide gets you from zero to a running iLoveReact app in five minutes. You will
scaffold a new Love2D project, edit the entry component, run it in watch mode, and
optionally produce a standalone binary. All you need is a completed installation
(see installation.txt).

=== EXAMPLES ===
Example 1: Scaffold and Run (Love2D)
---
# Scaffold a new project
ilovereact init my-app
cd my-app

# Start watch mode (rebuilds on file changes)
ilovereact dev

# In a second terminal, run Love2D
love .
---
Platforms: love2d

Example 2: Edit src/App.tsx
---
import { Box, Text, Pressable } from '@ilovereact/core';
import { useState } from 'react';

export default function App() {
  const [message, setMessage] = useState('Hello iLoveReact!');

  return (
    <Box style={{
      width: '100%',
      height: '100%',
      backgroundColor: '#0f172a',
      justifyContent: 'center',
      alignItems: 'center',
      gap: 20,
    }}>
      <Text style={{ fontSize: 32, color: '#f8fafc' }}>
        {message}
      </Text>
      <Pressable onPress={() => setMessage('You clicked!')}>
        {({ pressed }) => (
          <Box style={{
            backgroundColor: pressed ? '#2563eb' : '#3b82f6',
            paddingLeft: 24,
            paddingRight: 24,
            paddingTop: 12,
            paddingBottom: 12,
            borderRadius: 8,
          }}>
            <Text style={{ fontSize: 16, color: '#ffffff' }}>Click Me</Text>
          </Box>
        )}
      </Pressable>
    </Box>
  );
}
---
Platforms: All

Example 3: Build a Standalone Binary
---
# Lint first (catches layout errors before build)
ilovereact lint

# Take a screenshot to verify layout
ilovereact screenshot --output /tmp/preview.png

# Build self-extracting Linux binary
ilovereact build dist:love
---
Platforms: love2d

=== CRITICAL RULES ===
- Always use `ilovereact init` to scaffold projects. Do not mkdir and copy manually.
- Always use `ilovereact dev` for watch mode. Do not run esbuild --watch manually.
- Always use `ilovereact build` for production builds. Do not run esbuild manually.
- The CLI encodes correct esbuild flags, enforces lint gates, and handles runtime
  file placement.
- After modifying any component, run `ilovereact lint` to catch layout errors.
