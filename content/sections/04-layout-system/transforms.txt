=== OVERVIEW ===
iLoveReact supports a transform object on any element's style for applying geometric
transformations. Transforms affect the visual rendering of the element without changing its
position in the layout tree. A transformed element still occupies the same layout space it
would without the transform -- sibling elements are not affected.

The transform system supports translation, rotation, scaling, and skewing. Transform origin
defaults to the center of the element (0.5, 0.5) and can be adjusted with originX and
originY values ranging from 0 (top-left) to 1 (bottom-right). Transforms are commonly
combined with the animation system for motion effects.

=== API / SYNTAX ===
Transform style property:

  transform: {
    translateX?: number     // Horizontal offset in pixels
    translateY?: number     // Vertical offset in pixels
    rotate?: number         // Rotation in degrees (clockwise)
    scaleX?: number         // Horizontal scale factor (1 = no change)
    scaleY?: number         // Vertical scale factor (1 = no change)
    skewX?: number          // Horizontal skew in degrees
    skewY?: number          // Vertical skew in degrees
    originX?: number        // Transform origin X, 0-1 (default: 0.5 = center)
    originY?: number        // Transform origin Y, 0-1 (default: 0.5 = center)
  }

All transform sub-properties are optional. Only the properties you specify are applied.

Transform origin:
  originX and originY are normalized values from 0 to 1, where:
    0, 0 = top-left corner
    0.5, 0.5 = center (default)
    1, 1 = bottom-right corner

=== EXAMPLES ===
Example 1: Simple translation
---
<Box style={{
  width: 100,
  height: 100,
  backgroundColor: '#89b4fa',
  transform: { translateX: 50, translateY: 20 },
}} />
---

Example 2: Rotation from center
---
<Box style={{
  width: 100,
  height: 100,
  backgroundColor: '#f38ba8',

<Box style={{

---
<Box style={{
  width: 100,
  height: 100,
  backgroundColor: '#cba6f7',

Example 6: Animated transform with useSpring
---
import { Box } from '@ilovereact/core';
import { useSpring } from '@ilovereact/core';

function PulseBox({ active }: { active: boolean }) {
  const scale = useSpring(active ? 1.1 : 1.0, { stiffness: 200, damping: 15 });
  return (
    <Box style={{
      width: 100,
      height: 100,
      backgroundColor: '#89b4fa',
      transform: { scaleX: scale, scaleY: scale },
    }} />
  );
}
---
Platforms: All

=== PLATFORM NOTES ===
Love2D: Transforms are applied by painter.lua using Love2D's love.graphics.translate,
love.graphics.rotate, love.graphics.scale, and love.graphics.shear functions. The transform
is applied around the computed origin point before rendering the element.

Web: The transform object is converted to a CSS transform string by styleToCSS(). For example,
{ translateX: 50, rotate: 45, scaleX: 2 } becomes "translate(50px, 0px) rotate(45deg) scale(2, 1)".
Transform origin is converted from 0-1 normalized values to percentage values for CSS
transformOrigin.

Grid targets: Transforms have limited applicability in character-grid contexts. Translation
may shift rendered position by character cells. Rotation and scaling are generally not
supported on character-grid targets.

=== CRITICAL RULES ===
- Transforms do NOT affect layout -- the element still occupies its original layout rectangle
- Transform origin defaults to center (0.5, 0.5), not top-left
- Rotation is specified in degrees, not radians
- Scale values of 1.0 mean no change -- 2.0 doubles the size, 0.5 halves it
- Transforms are visual-only and commonly used with the animation system for motion effects
- OriginX/originY are 0-1 normalized, not pixel values
